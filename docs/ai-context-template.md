# AI context template

Use this file as `docs/ai-context.md` in a repository when you want one shared
AI-facing document for `codex`, `gh copilot`, and human contributors.

## Project snapshot

- What the project does in one or two sentences
- The main runtime, framework, and package manager
- The most important directories

## Bootstrap

- Install command
- Dev server command
- Test command
- Lint / format command
- Build command

## Working rules

- Files or modules that need extra care
- Review expectations
- Performance or security constraints
- What should never be changed casually

## Validation

- Minimum checks before opening a PR
- When manual verification is required
- Known flaky checks or slow suites

## Notes for AI tools

- Preferred place for task-specific instructions
- Whether networked tools are allowed
- Whether generated files should be committed
